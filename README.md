# BusyKitchen

BusyKitchen は、料理の注文と調理を題材にしたアプリケーションです。  
Javaのマルチスレッド学習として作成しました。 

注文はキューに追加され、複数のコックが並行して料理を作ります。  
料理ごとに調理時間が異なるため、注文順と完成順が一致しないことがあります。  

## アプリケーション概要

- 注文をキューに登録する
- 複数のコックが別スレッドで動く
- コックはキューから注文を取り出して調理する
- 調理時間が異なるため、同時進行の様子を確認できる

## 主な学習ポイント

- `Thread` の基本的な使い方
- `Runnable` を使った処理の分離
- `BlockingQueue` を使ったスレッド間連携
- 複数スレッドによる同時実行
- `join()` による終了待ち
- 終了シグナルによるスレッド停止

## クラス構成

### `BusyKitchenMain`
アプリケーションの起動クラスです。  
注文キューを作成し、コックを起動し、注文を登録します。

### `Order`
1件の注文を表すレコードです。  
注文番号と、注文されたメニューを持ちます。

### `MenuItem`
料理メニューを表す列挙型です。  
料理名と調理時間を管理します。

### `Cook`
コックを表すクラスです。  
`Runnable` を実装し、注文キューから料理を取り出して調理します。

### `Customer`
お客さんを表すクラスです。  
`Runnable` を実装し、注文キューに希望する料理を入れます。


## 実行例

```text
=== BusyKitchen 開店 ===
Cook-A が調理開始: 注文1 ラーメン
Cook-B が調理開始: 注文2 餃子
Cook-B が調理完了: 注文2 餃子
Cook-B が調理開始: 注文3 チャーハン
...
=== BusyKitchen 閉店 ===
```
